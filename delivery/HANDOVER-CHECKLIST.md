# HAJR A° Academy — Handover Checklist

Owner ticks each box. Once all 10 are confirmed, the partnership
agreement (20% rev-share) is signed and the support tier kicks in.

---

- [ ] **Vercel deployment access transferred** — project ownership moved
      to the academy's Vercel team (or owner is invited as Owner-tier
      member). Billing card updated to academy account.

- [ ] **Supabase project access transferred** — owner added as Owner of
      the project; original developer kept as Developer member for
      ongoing maintenance.

- [ ] **Resend API access transferred** — domain verified under academy
      Resend workspace; sending domain `mail.hajracademy.sa` (or your
      chosen) listed as Verified.

- [ ] **Zoom app access transferred** — Server-to-Server OAuth app moved
      to academy Zoom workspace; webhook URL points to production
      `/api/zoom/webhook`.

- [ ] **Domain DNS access transferred** — registrar login handed over;
      DNS records (CNAME → Vercel, TXT → Resend domain verification)
      documented in a secure note.

- [ ] **Admin credentials reset and rotated** — `AUTH_SECRET`,
      `RESEND_API_KEY`, `ANTHROPIC_API_KEY`, `ZOOM_CLIENT_SECRET`,
      `CRON_SECRET` all rotated in Vercel env vars + redeployed.

- [ ] **Brand book PDF received** — owner has the 10-page brand book
      either downloaded from `/admin/brand-kit → Download Complete
      Brand Book` or emailed via the page form. Verified the PDF
      opens and prints cleanly.

- [ ] **Validation meeting completed** — went through all 12 tabs at
      `/admin/validation` with the teaching team, ticked each verified
      box, exported the validation report PDF.

- [ ] **Partnership agreement (20% rev-share) signed** — paper or
      DocuSign — original retained by both parties.

- [ ] **Support tier acknowledged** — owner has read README §8 and
      understands what is and isn't covered free-of-charge post-handover.
